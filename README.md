# Clean Text 🧼

Clean Text는 의료진과 의학도를 위한 의료 텍스트 보조 도구입니다.  
복잡하고 비구조화된 EMR 텍스트를 더 읽기 쉽게 정리하고, row-based 검사 결과를 구조화된 표로 변환하여 엑셀로 내보낼 수 있도록 돕습니다.

이 프로젝트는 완전한 EMR parser나 판독 엔진이 아니라, 반복적인 수작업을 줄여주는 practical workflow tool을 목표로 합니다.

---

## 주요 기능

### 1. 텍스트 클리너 (Text Cleaner)

EMR 특유의 공백, 줄바꿈, 섹션 구조를 정리하여 가독성을 높입니다.

지원 기능 예시:
- 전각 공백 제거
- 연속 공백 / 빈 줄 정리
- 섹션 헤더 표준화
- Problem list / note block 구조 정리
- 보고서형 텍스트(내시경, 영상, 병리 등) 가독성 개선

이 기능은 표 변환보다 **읽기 좋게 정리하는 것**에 초점이 맞춰져 있습니다.

---

### 2. 테이블 변환기 (Lab to Table)

텍스트 형태의 검사 결과를 파싱하여 구조화된 표로 변환합니다.

지원 범위:
- 정량 검사
  - CBC
  - chemistry
  - blood gas
  - coagulation
  - 일부 point-of-care 검사
- 정성 / 반정량 검사
  - urine dipstick
  - urine microscopy
  - Negative / Positive / Trace / 1+ / 2+ / 3+
  - cells/HPF, /LPF, range/threshold 표현

출력 기능:
- 엑셀(.xlsx) 다운로드
- 시트별 결과 분리
- 그룹화된 표 생성
- 원문 / 미파싱 줄 / 보고서형 텍스트 보존

---

## 입력 분류 방식

Clean Text의 테이블 변환기는 입력을 처음부터 세 가지로 나누어 처리합니다.

### 1. 정량 검사
숫자값, 단위, 참고치가 비교적 명확한 검사입니다.

예:
- WBC 7.69 ×10³/㎕ 4~10
- Creatinine 7.09 mg/dL 0.6~1.1
- Na 138 mmol/L 135~145

처리 방식:
- 검사명 / 결과값 / 단위 / 참고치 구조로 정리
- 원문에 ▲ / ▼가 있으면 그대로 반영
- 엑셀에서도 상승/하강 색 표시 가능

### 2. 정성 / 반정량 검사
문자형 결과나 반정량 표현을 사용하는 검사입니다.

예:
- Negative / Positive / Trace
- 1+ / 2+ / 3+
- 1~3 cells/HPF
- 100이상 cells/HPF

처리 방식:
- 결과값과 참고치를 바탕으로 이상 여부를 판정
- 기준보다 높거나 비정상이면 빨간색
- 기준보다 낮으면 파란색
- 판정 불가한 경우는 기본색 유지

### 3. 보고서형 검사
장문 서술형 결과입니다.

예:
- 내시경 결과
- 영상 판독 결과
- 병리 보고서
- narrative 중심의 미생물/혈액은행 결과

처리 방식:
- Lab to Table에서 구조화하지 않음
- 별도 `Ignored_Reports` 시트로 분리
- Text Cleaner 사용 권장

즉,  
**숫자 중심 결과는 테이블 변환기**,  
**서술형 결과는 텍스트 클리너**  
라고 생각하시면 가장 편합니다.

---

## 엑셀 출력 구성

입력 내용에 따라 다음과 같은 시트가 생성될 수 있습니다.

- `Parsed_Labs`
  - 정량 검사 결과
- `Qualitative_Results`
  - 정성 / 반정량 검사 결과
- `Grouped_Tables`
  - 정량 결과를 보기 좋게 그룹화한 시트
- `Grouped_Qualitative`
  - 정성 결과를 그룹화한 시트
- `Original_Text`
  - 원문 보관
- `Unparsed`
  - 파싱되지 않은 줄
- `Ignored_Reports`
  - 보고서형으로 분류되어 표 변환에서 제외된 텍스트

이 구조를 통해 결과를 바로 사용하는 것뿐 아니라,
파싱 실패나 원문 대조도 함께 할 수 있습니다.

---

## 정성 검사 판정 방식

정성 / 반정량 결과는 결과값과 참고치를 기반으로 규칙적으로 판정합니다.

예:
- `Positive`, `Trace`, `1+`, `2+`, `3+` 등은 비정상으로 해석 가능
- `Negative` 기준에서 벗어나는 결과는 빨간색
- `<50`, `>50`, `30~40`, `0~3 cells/HPF` 같은 참고치가 있으면 그 기준에 따라 비교
- 기준보다 높으면 빨간색, 낮으면 파란색

즉, 정성 검사도 가능한 범위 내에서 자동 강조 표시를 지원합니다.

## 사용 가이드

### 테이블 변환기를 먼저 쓰면 좋은 경우
- CBC / chemistry / ABG 같은 row-based 검사 결과
- urine dipstick / microscopy 결과
- 엑셀로 바로 정리하고 싶은 lab block

### 텍스트 클리너를 먼저 쓰면 좋은 경우
- 내시경 / 영상 / 병리 결과
- 줄바꿈과 공백이 많이 깨진 텍스트
- 읽기 좋게 정리하는 것이 우선인 경우

---

## 현재 한계

Clean Text는 규칙 기반 parser이므로, 다음과 같은 한계가 있습니다.

- 병원 / 부서별 포맷 차이에 따라 일부 항목은 완전히 파싱되지 않을 수 있음
- 단위와 참고치가 한 줄에 붙어 있는 경우 예외 처리가 필요할 수 있음
- 장문 보고서형 결과는 표 구조화보다 텍스트 정리가 더 적합함
- 중요한 데이터는 항상 원문과 함께 확인하는 것이 좋음

즉 이 도구는 원문 확인을 완전히 대체하기보다,
정리 / 구조화 / 엑셀 변환을 빠르게 보조하는 데 목적이 있습니다.
